Homework 2: Gemini Cookbook

Your goal is to develop a small proof-of-concept NLP application using a generative AI model, inspired by Google's Gemini API Cookbook examples. The focus is on learning to use a generative language model in a practical application, integrating it with other tools or data, and documenting your process.

Objectives

By the end of this homework, you will be able to:

  1. Explore and understand the Gemini Cookbook examples for generative AI applications
  2. Design a proof-of-concept NLP application that leverages a generative model
  3. Craft effective prompts and manage the model’s outputs to address a specific NLP task
  4. Integrate an external API or data source (e.g., web search, dataset retrieval, or code execution) to enrich your application
  5. Document your project design, implementation challenges, and results in a structured report
  6. Submit both your code (e.g., a Python notebook) and the written report

Instructions

  1. Explore the Gemini Cookbook Examples: Start by reviewing the Gemini API Cookbook examples. These examples illustrate how to combine multiple AI features or third-party tools to build complex applications.
    • Notice how each example uses the Gemini generative model for NLP tasks (e.g., text generation, answering questions)
    • Identify any additional features or integrations, such as using a search engine for grounding facts or executing code in response to a prompt
    • Tip: You do not need to run all the code – even reading through the examples and their descriptions can give you insight
  2. Choose Your Project Idea: Based on what you observed in the examples, think of a small NLP application you would like to build. It should involve generative AI (e.g., using the Gemini API or a similar language model) and ideally one additional integration (another tool, dataset, or API). Feel free to be creative – the goal is to come up with your own twist rather than copying an example.
  3. Plan Your Solution:
    • Input/Output: Define what input the user will provide and what output the application should generate.
    • Role of Generative AI: Decide how you will use the generative model (e.g., answering questions, creating content, summarizing text). Sketch out example prompts you might use.
    • Integration (if any): Determine which external tool or data you might need. This might include:
      • An API call to fetch information (like a web search or database query).
      • Utilizing a library or tool that complements the generative AI (e.g., for code execution).
    • Flow: Map out a step-by-step process for your application. For example: "User asks a question → App fetches related data → Generative model creates a final answer."
  4. Implement the Application:
    • Use the Gemini API via the provided SDK (or an alternative model) to implement the core functionality.
    • Develop your solution according to your plan. Keep your code simple, clear, and well-commented.
    • Document key steps, such as API calls to the generative model and any integrations, directly in your code.
    • If necessary, refine your prompt or add post-processing steps to manage the model's output.
  5. Test and Refine:
    • Run your application on several sample inputs to ensure it behaves as expected.
    • Analyze the outputs and adjust your approach if the results are not as intended.
  6. Document Your Work: Write a brief report following the Documentation Requirements below. Explain what you built, how it works, and discuss your results.
  7. Submission: Submit both your code (as a Python script or notebook) and your written report. Ensure your code is runnable and include clear instructions, without hard-coding API keys.

Project Scope and Requirements

  • Complexity: The project should be more involved than a single API call but limited in scope.
  • Use of Generative AI: Your application must use a generative model for a core part of the functionality.
  • Integration of Features/APIs: Include at least one integration, such as an external data source/API or a secondary feature (like function calling or code execution).
  • Use of Existing Code: You may reuse snippets from the Gemini Cookbook for reference but make sure to cite.
  • Collaboration: Working with partner is encouraged. Submit the work separately.
  • Submission: Notebook URL + Written Report (word doc or google doc)

Documentation Requirements (Report Template)

Your report should follow this structure:

  1. Title and Overview: Provide a project name and a one-paragraph overview explaining what the application does.
  2. Problem Statement and Motivation: Describe the problem you are addressing and why it is useful in NLP.
  3. Inspiration from Gemini Cookbook Examples: Mention which examples or features influenced your design. Cite specific examples or URLs.
  4. Methodology and Approach:
    • Explain how you built the application and how the generative model was used
    • Describe any additional integrations and the overall workflow
  5. Implementation Details:
    • Highlight important aspects of your implementation
    • Explain how you handled the model’s response and any challenges you encountered
  6. Results and Example Outputs:
    • Show sample inputs and outputs, and discuss the accuracy or usefulness of the results.
  7. Discussion (Challenges and Reflections):
    • Reflect on challenges faced, how you addressed them, and what you learned during development.
  8. Conclusion and Future Work:
    • Summarize the project outcome and suggest possible improvements or extensions.
  9. References:
    • List any sources, documentation, or code examples you consulted.

Example Project Ideas

  • Text Document Analyzer (Structured Output): A tool that takes a document and extracts key information or summarizes it (for example, extracting invoice details or summarizing a research paper).
  • Code Generator and Executor: A helper that generates Python code based on a natural language request (e.g., solving math problems or plotting sequences), then executes the code.
  • Interactive Storyteller or Tutor: An application that engages in a multi-turn interaction to generate creative storytelling or educational responses.
  • Email Drafting Assistant (with Template Integration): A tool that drafts emails from bullet points or short descriptions, optionally integrating tone/style choices.

Grading Criteria

  • Generative AI Usage (25%): Effective use of the generative model for NLP, including prompt formulation and handling the model's output.
  • Integration of API/Tools (20%): Proper and meaningful integration of external data sources or features that complement the generative AI functionality.
  • Functionality and Completeness (25%): The application works as intended, with all main components implemented and tested.
  • Creativity and Originality (10%): Innovative ideas and a unique approach to the problem.
  • Code Quality and Best Practices (10%): Well-organized, readable code with proper comments and error handling.
  • Documentation and Report (10%): A clear, well-structured report explaining the project, challenges faced, and the overall process.

Note: The majority of the grade (over 70%) is based on the effective use of generative AI, integration, and overall functionality. Ensure these aspects are thoroughly addressed in your project.
